# 📚 Centro de Documentación - ÓrbitA

Bienvenido a la fuente oficial de conocimiento técnico y funcional de la plataforma **ÓrbitA**. Este repositorio de documentos ha sido diseñado para proporcionar una visión clara, estructurada y técnica del sistema, facilitando su mantenimiento, escalabilidad y comprensión por parte del equipo de desarrollo.

> [!IMPORTANT]
> Esta carpeta `/docs` es la **Fuente Oficial de Conocimiento** del proyecto. Cualquier modificación estructural o funcional en la aplicación **debe** verse reflejada en estos documentos.

## 🗂️ Índice de Documentación

A continuación se detallan los documentos disponibles en este centro de conocimiento:

1.  **[Arquitectura del Proyecto](./ARQUITECTURA_PROYECTO.md)**
    *   Descripción de la arquitectura general del sistema (Spring Boot + Thymeleaf).
    *   Principios de diseño, capas de lógica y flujo de datos.

2.  **[Estructura del Proyecto](./ESTRUCTURA_PROYECTO.md)**
    *   Mapa detallado de carpetas y archivos.
    *   Propósito de cada paquete Java y directorio de recursos.

3.  **[MVC y Flujo de Vistas](./MVC_Y_FLUJO_DE_VISTAS.md)**
    *   Explicación del patrón Model-View-Controller implementado.
    *   Funcionamiento de la navegación, layouts y carga dinámica de fragmentos Thymeleaf.

4.  **[Seguridad y Roles](./SEGURIDAD_Y_ROLES.md)**
    *   Detalle técnico de Spring Security y RBAC (Control de Acceso Basado en Roles).
    *   Matriz de permisos, normalización de strings y fuente de verdad en Firebase.

5.  **[Módulos del Sistema](./MODULOS_DEL_SISTEMA.md)**
    *   Inventario funcional de los módulos (Operativos, Analíticos, Predictivos, etc.).
    *   Propósito y alcance de cada componente de la plataforma.

6.  **[Páginas y Funciones](./PAGINAS_Y_FUNCIONES.md)**
    *   Descripción detallada página por página.
    *   Objetivos funcionales, acciones permitidas y dependencias técnicas.

7.  **[Reglas de Documentación](./REGLAS_DE_DOCUMENTACION.md)**
    *   Protocolo obligatorio para la actualización continua de esta documentación.
    *   Garantía de sincronización entre el código y su descripción técnica.

---

## 🚀 Por dónde empezar
Si eres nuevo en el proyecto, te recomendamos seguir este orden de lectura:
1.  **Arquitectura del Proyecto**: Para entender el "Big Picture".
2.  **Seguridad y Roles**: Para comprender cómo se restringe el acceso.
3.  **Estructura del Proyecto**: Para ubicarte dentro del código fuente.
4.  **Páginas y Funciones**: Para conocer la operatividad real de la aplicación.
